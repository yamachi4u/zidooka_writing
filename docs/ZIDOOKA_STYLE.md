# ZIDOOKA! Writing Style Guidelines

When asked to "Write in ZIDOOKA! style" (ZIDOOKA!で書いて), follow these rules:

## 1. Content Structure & Research
- **PREP Method:** Use the PREP (Point, Reason, Example, Point) structure implicitly. Do not explicitly label sections as "Point" or "Reason".
- **Comprehensive Research:** Search extensively and cover the topic comprehensively.
- **SERP Analysis:** Analyze Search Engine Results Pages (SERP) and create content that can outperform competitors.
- **Environment Details:** If the user provides specific environment details (e.g., "I encountered this on Windows 11"), include them.
- **EEAT:** Focus heavily on Experience, Expertise, Authoritativeness, and Trustworthiness.

## 2. Tone & Style
- **Polite Tone:** Use "Desu/Masu" (です・ます) style.
- **No Emojis:** Avoid emojis to maintain a professional, AI-like (but high-quality) tone.
- **Varied Intros:** While the internal structure is PREP, vary the tone, introduction, and writing style for each article. Do not use a fixed template for the opening.
- **First Person:** Use phrases like "In this article..." (この記事では～) rather than "ZIDOOKA concludes..." or "ZIDOOKA thinks...".

## 3. Formatting & Links
- **No Canvas References:** Do not mention "Canvas" or "ChatGPT".
- **Reference URLs:**
    - List reference URLs at the very end of the article.
    - Format: Numbering, Title, Raw URL.
    - **Do NOT use Markdown links** (e.g., `[Title](URL)`). Use raw URLs.
    - **Remove Tracking Parameters:** Delete `?utm_source=chatgpt.com` etc.
    - **Article-Level Links:** Link to specific articles, not top-level domains.
    - **Bilingual:** Include the URL list in both Japanese and English versions.

## 4. Output Format
- **WordPress Ready:** The output must be ready to be pasted into WordPress (Markdown format).
- **Bilingual:** Create both Japanese and English versions.
