# 403 Forbidden エラーの解決方法 (WAF)

画像のアップロードや記事の投稿時に `403 Forbidden` エラーが発生しています。
これは、レンタルサーバー（ロリポップやXserverなど）の **WAF (Web Application Firewall)** が、API経由の通信を「攻撃」と誤検知してブロックしているためです。

## 解決手順

1.  **レンタルサーバーの管理画面** にログインする。
2.  **セキュリティ** または **WAF設定** のメニューを開く。
3.  **ログ** を確認し、直近（数分前）にブロックされた通信があるか見る。
    *   URLに `/wp-json/wp/v2/media` や `/wp-json/wp/v2/posts` が含まれているはずです。
4.  その通信（シグネチャ）を **「許可」** または **「除外」** 設定にする。
    *   ※WAF全体を無効にするのはセキュリティ上おすすめしませんが、一時的にOFFにして投稿できるか試すのはアリです。
5.  設定反映まで数分〜数十分かかる場合があるので、少し待ってから再度コマンドを実行する。

## 補足
特に「画像のアップロード」や「英語の記事（SQLインジェクションと誤認されやすい）」で引っかかることが多いです。
