# Claude Opus 4.7を使ってみた体感：軽いけど、深く任せるには少し不安

![Claude Opus 4.7の体感メモ](../images/2026/claude-opus-4-7-light-review-thumbnail.png)

ClaudeにOpus 4.7があったので少し使ってみました。

最初の印象は、正直に言うと「軽い」です。悪い意味だけではありません。返答のテンポはよく、雑談や軽い壁打ちには向いていそうです。ただ、コーディングや仕様整理のように、こちらが深く考えてほしい場面では少し物足りなさを感じました。

この記事は、公式スペックを細かく比較する記事ではなく、実際に触ったときの体感メモです。

:::conclusion
Opus 4.7は会話相手としては扱いやすい一方で、重い開発作業を丸ごと任せるモデルとしては、現時点の体感では少し軽すぎます。
:::

## Opus 4.6より軽い

以前使っていたOpus 4.6は、かなり「重い」印象がありました。

ここでいう重いとは、単に遅いという意味だけではありません。考え込む感じがあり、返答にも圧がある。複雑なコードや長い仕様を渡したときには頼もしさがある一方で、軽い相談をしたいだけのときには少し大げさに感じる場面もありました。

それに比べると、Opus 4.7は明らかに軽く感じます。

会話のテンポはよく、こちらの雑な話にも返ってくる速度が速い。重苦しさは減っています。ちょっとした雑談や、「この方向どう思う？」くらいの相談なら、むしろ4.7のほうが使いやすいかもしれません。

## ただ、考えている感じは薄い

気になったのは、深く考えている感じがあまりないことです。

こちらが雑に話したときに、雑なまま返ってくる。もちろんそれが良い場面もあります。しかし、仕事でAIに期待しているのは、雑な入力を受け取ったあとに、裏側で構造化して、論点を整理して、危ない部分を拾ってくれることです。

Opus 4.7は、その踏み込みが少し浅く感じました。

たとえば、文章の方向性を軽く相談するだけなら問題ありません。けれど、既存の作業フローに乗せて記事化する、コードベースを読んで修正する、仕様の矛盾を見つける、といった場面では「そこで止まるのか」と感じることがあります。

:::note
軽快さはメリットですが、実務では「軽い返答」と「浅い判断」の境目がかなり重要です。
:::

## 雑談には向いている

Opus 4.7の良いところは、雑談しやすいことです。

AIモデルに対して、いつも重い推論や完璧な実装だけを求めているわけではありません。ちょっとした違和感を言語化したいとき、アイデアの種を出したいとき、考えを転がしたいときには、軽く返ってくるモデルのほうが使いやすいこともあります。

そういう意味では、Opus 4.7は会話相手としては悪くありません。

Opus 4.6のように、こちらの軽い話にも重く構えて返ってくる感じが少ないので、雑に話しかけやすい。AIを作業パートナーというより、軽い壁打ち相手として使うなら相性は良さそうです。

## 仕事で使うなら用途を分けたい

今の体感では、Opus 4.7を何でも任せる主力モデルにするのは少し不安です。

特に、以下のような作業では重めのモデルを使いたいです。

- 既存コードを読んで修正する
- 複数ファイルにまたがる原因調査をする
- 記事の構成を詰めて品質を上げる
- 仕様の抜けや矛盾を見つける
- 投稿前の細かい表現を整える

一方で、以下の用途ならOpus 4.7でも十分使えそうです。

- 雑談
- 軽い相談
- アイデア出しの初手
- 違和感の言語化
- 文章の方向性をざっくり見る

:::example
自分なら、最初の壁打ちはOpus 4.7、実装や最終判断はもっと重いモデル、という使い分けにします。
:::

## 「軽い」は悪いことではない

今回の感想を一言でいうと、「Opus 4.7は劣化した」というより、「性格が軽くなった」という印象です。

ただし、こちらがAIに求めているものによって、その軽さは評価が変わります。雑談相手として見れば扱いやすい。日常的なメモや軽い相談では、むしろちょうどいい可能性があります。

でも、開発や記事制作のような作業では、軽さがそのまま不安になります。

AIに任せたいのは、単に返答することではなく、こちらの雑な意図を拾って、必要なところで立ち止まり、危ない部分を指摘して、作業を前に進めることです。その点で、Opus 4.7はもう少し深く踏み込んでほしい場面がありました。

:::conclusion
Opus 4.7は雑談や初期の壁打ちには良いですが、仕事の中核を任せるなら、まだ重めのモデルと使い分けたほうが安心です。
:::
