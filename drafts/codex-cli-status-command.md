OpenAIのCodex CLIを使って開発を効率化している方、残りのクレジットや使用状況をサッと確認したいですよね。

Codex CLIでは、セッション中に`/status`と入力するだけで、現在の設定やトークン使用状況、クレジットの残量などが確認できます。

この記事では、`/status`コマンドの使い方と表示される情報の意味を徹底解説します。

## `/status`コマンドとは

`/status`はCodex CLIのスラッシュコマンドの一つで、現在のセッション情報を一覧表示するコマンドです。

【結論】`/status`コマンドを使うと、アクティブなモデル、承認ポリシー、書き込み可能なディレクトリ、残りのトークン容量などが一目で確認できます。

## 基本的な使い方

Codex CLIの対話モード中に、以下のように入力するだけです。

```
/status
```

プロンプト欄で `/` を入力するとスラッシュコマンドのポップアップが表示されるので、そこから `status` を選択してもOKです。

## 表示される情報の解説

`/status`コマンドを実行すると、以下のような情報が表示されます（実際の出力は環境によって異なります）。

### アクティブなモデル

現在使用中のモデルが表示されます。例：`gpt-5.2-codex`、`gpt-4.1-mini`など。

セッション中にモデルを切り替えたかどうかの確認に便利です。

### 承認ポリシー（Approval Policy）

コマンド実行時の承認設定が表示されます。

- `Auto`：自動承認
- `On Request`：リクエスト時に承認
- `Read Only`：読み取り専用

【注意】設定が想定と異なると、意図しないコマンドが実行される可能性があります。

### 書き込み可能なディレクトリ（Writable Roots）

Codexが書き込みを許可されているディレクトリの一覧です。デフォルトではワークスペース内のディレクトリが含まれます。

### トークン使用状況

- 現在のトークン使用量
- コンテキストウィンドウの残り容量

これにより、会話が長くなりすぎてトークン上限に近づいているかどうかを判断できます。

### レートリミットとクレジット

アカウントの利用制限に関する情報も表示されます。

- 残りのクレジット
- 現在のレートリミット状況
- 使用可能な5時間ブロックの残量

これらの情報は、課金プランや利用状況に応じて変動します。

## `/status`を活用するシーン

### 1. 予期せぬ挙動が起きたとき

モデルや承認ポリシーが意図した設定になっているか確認できます。

### 2. トークンが足りなくなったとき

長時間のセッションでトークン消費が増えてきたら、`/compact`コマンドと併用して対話を要約し、コンテキストを節約できます。

### 3. レートリミットに達しそうなとき

クレジットの残量が少ないとわかれば、優先度の高いタスクに集中できます。

## その他の便利なスラッシュコマンド

`/status`と併せて覚えておくと便利なコマンドを紹介します。

| コマンド | 説明 |
| --- | --- |
| `/model` | 使用するモデルを切り替える |
| `/approvals` | 承認ポリシーを変更する |
| `/compact` | 会話を要約してトークンを節約 |
| `/diff` | Gitの差分を表示 |
| `/new` | 新しい会話を開始 |

## まとめ

`/status`コマンドは、Codex CLIを効果的に使うために欠かせないツールです。

【対処】セッションの途中で定期的に`/status`を実行して、現在の状況を把握する習慣をつけましょう。

トークン使用量やクレジット残量を確認しながら作業を進めることで、予期せぬ制限に引っかかることを防げます。
