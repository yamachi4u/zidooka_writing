チラシを何万部配布しても、それがどれだけWebサイトへのアクセスにつながったのかを計測できなければ、次の施策の判断材料になりません。オフライン施策とオンライン計測の溝を埋める方法として、UTMパラメータ付きURLをQRコードにする手法があります。

この記事では、[UTM生成ツール](https://tools.zidooka.com/jp/utmtools/generator)と[QRコード生成ツール](https://tools.zidooka.com/jp/utmtools/qr-generator)を組み合わせて、チラシ・名刺・店頭POPなどのオフライン媒体からの流入をGA4で正確に計測する方法を解説します。

## なぜオフライン施策の計測が難しいのか

オンライン広告であれば、Google広告やFacebook広告のプラットフォームがクリック数やコンバージョンを自動で計測してくれます。しかしオフラインの印刷物にはクリックという概念がありません。

QRコードを使わずに印刷物にURLだけを記載した場合、ユーザーがそのURLを手入力してアクセスすると、GA4では「直接アクセス（direct）」として記録されます。これではチラシ経由のアクセスなのか、ブックマークからのアクセスなのか区別がつきません。

【結論】UTMパラメータ付きURLをQRコードにすることで、オフライン媒体ごとの流入を個別にトラッキングできるようになります。

## 実践手順

### ステップ1: UTMパラメータの設計

まず、どの媒体・施策を計測したいかを整理し、パラメータの値を決めます。

:::example
春の新商品チラシの場合：
- utm_source: `flyer`（チラシ）
- utm_medium: `print`（印刷物）
- utm_campaign: `spring_new_product_2026`（キャンペーン名）
- utm_content: `front_page`（チラシの表面に掲載）
:::

同じキャンペーンで複数の媒体を使う場合は `utm_source` で媒体を区別します。

| 媒体 | utm_source | utm_medium | utm_campaign |
|---|---|---|---|
| チラシ | flyer | print | spring_2026 |
| 名刺 | business_card | print | spring_2026 |
| 店頭POP | store_pop | print | spring_2026 |
| イベント配布物 | event_handout | print | spring_2026 |
| 製品パッケージ | package | print | spring_2026 |

### ステップ2: UTM付きURLの生成

ZIDOOKA Toolsの[UTM生成ツール](https://tools.zidooka.com/jp/utmtools/generator)を使ってURLを作成します。

:::step
1. UTM生成ツール（<https://tools.zidooka.com/jp/utmtools/generator>）を開く
2. ランディングページのURLを入力する
3. 設計したパラメータの値を各フィールドに入力する（入力候補から選択も可能）
4. 生成されたURLをコピーする
:::

### ステップ3: QRコードの生成

生成されたURL欄の下にある「QRコードを生成」ボタンをクリックすると、QRコード生成ツールにURLが自動で引き継がれます。

:::step
1. QRコード生成ツールでUTMパラメータが検出されたことを確認する
2. 印刷サイズに合わせてQRコードのサイズを選択する（印刷用はL以上推奨）
3. 必要に応じて色を変更する
4. 「PNGダウンロード」でファイルを保存する
:::

### ステップ4: 印刷物への配置

ダウンロードしたQRコードをデザインデータに配置します。

:::warning
QRコードの周囲には「クワイエットゾーン」と呼ばれる余白が必要です。QRコードの辺の長さの10%以上の余白を確保してください。背景画像やテキストがQRコードに重ならないように注意します。
:::

QRコードの近くに「詳しくはこちら」「スマホでスキャン」などの誘導文を添えると、スキャン率が上がります。

### ステップ5: GA4でデータを確認する

チラシ配布後、GA4のレポートでデータを確認します。

GA4の「レポート」 >「集客」>「トラフィック獲得」を開き、ディメンションを「セッションのソース / メディア」に設定すると、`flyer / print` のような形でオフライン媒体からの流入が表示されます。

「セッションのキャンペーン」ディメンションを追加すれば、キャンペーン別の比較もできます。

## 複数媒体のA/Bテスト

UTMパラメータを活用すれば、同じキャンペーンで異なる媒体やデザインの効果を比較できます。

:::example
チラシのデザインAとデザインBを比較する場合：
- デザインA: `utm_content=design_a`
- デザインB: `utm_content=design_b`

それぞれ異なるQRコードを生成して配布すれば、GA4の `utm_content` ディメンションでどちらのデザインがより多くのアクセスを獲得したかを比較できます。
:::

配布場所による効果の違いを見たい場合は `utm_term` にエリア名を入れる方法もあります。

## 運用のポイント

### 命名規則を統一する

チーム内でパラメータの命名規則を決めておかないと、同じ施策が `flyer` と `Flyer` と `チラシ` の3つの値で記録されてしまいます。

:::note
GA4はパラメータの値の大文字小文字を区別します。すべて小文字、区切り文字はアンダースコアに統一するのが一般的です。
:::

### 配信前にUTM解析ツールで検証する

QRコードを印刷する前に、[UTM解析ツール](https://tools.zidooka.com/jp/utmtools/parser)にURLを貼り付けて、パラメータが正しく設定されているかを確認します。印刷物は修正が効かないため、この検証ステップは重要です。

### QRコードのテストスキャン

印刷前にスマートフォンで実際にQRコードをスキャンし、正しいURLが開くことを確認してください。特に短縮URLを経由する場合は、リダイレクト先が正しいかもチェックします。

## まとめ

UTMパラメータとQRコードを組み合わせることで、これまでブラックボックスだったオフライン施策の効果をGA4で数値として把握できるようになります。チラシ、名刺、店頭POP、イベント配布物など、印刷物を使うあらゆる場面で活用できる手法です。
