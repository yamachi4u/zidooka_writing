Discordに新しいアプリ（Bot）の導入形式が登場して、「今までと何が違うの？」と戸惑っている方もいるかもしれません。

これまでは「サーバーの管理者がBotを招待する」のが一般的でしたが、新しく「**User Install（ユーザー導入）**」という仕組みが加わりました。

この記事では、従来の「Guild Install（サーバー導入）」と新しい「User Install（ユーザー導入）」の違いを、初心者の方にもわかりやすく解説します！

## 1. Guild Install（サーバー導入）とは？

これまでのDiscordで一般的だった、伝統的なBotの導入方法です。

- **対象**: 特定のDiscordサーバー（ギルド）
- **誰が使う？**: サーバーに参加している**メンバー全員**
- **主な用途**:
    - サーバーの管理・運営（荒らし対策など）
    - 読み上げBot
    - サーバー内でみんなで遊ぶゲーム機能など
- **権限の設定**: サーバーの「ロール」機能を使って、Botができることを管理者が細かく設定します。

**ポイント:**
「サーバーに住まわせる」イメージです。管理者が招待しないと使えませんし、そのサーバーの外では使えません。

## 2. User Install（ユーザー導入）とは？

新しく登場した、より個人的に便利に使える形式です。

- **対象**: あなた自身のDiscordアカウント
- **誰が使う？**: **あなた個人**（導入した本人）
- **主な用途**:
    - 自分専用の便利ツール（計算機、翻訳など）
    - 個人のメモ帳・ノート機能
    - どのサーバーにいても使えるスタンプや機能
- **設定場所**: Discord Developer Portalの「Installation」タブなどで設定可能です。
- **特徴**: DM（ダイレクトメッセージ）や、あなたが参加している**どのサーバーからでも**呼び出して使うことができます。

**ポイント:**
「スマホにアプリを入れる」イメージに近いです。自分がどこに行っても（どのサーバーにいても）、そのアプリの機能を持ち歩くことができます。

## まとめ：違いを比較！

| 特徴 | Guild Install (サーバー導入) | User Install (ユーザー導入) |
| --- | --- | --- |
| **対象** | サーバー (場所) | アカウント (人) |
| **使える人** | サーバーのメンバー全員 | 自分だけ |
| **使える場所** | Botがいるサーバー内のみ | DMや参加している全サーバー |
| **主な用途** | 運営、管理、コミュニティ機能 | 個人用ツール、メモ、便利機能 |
| **誰が入れる？** | サーバー管理者 | 各ユーザー自身 |

**Guild Install**は「みんなのための設備」、**User Install**は「自分だけの道具箱」と覚えるとわかりやすいですね！

用途に合わせて使い分けて、より快適なDiscordライフを送りましょう。
